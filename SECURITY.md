# 安全政策

## 支持范围

StoryMaker 目前处于早期开发阶段，安全修复优先覆盖 `main` 分支和最新发布包。历史实验版本不承诺单独维护。

## 报告漏洞

如果你发现安全问题，请优先使用 GitHub 的私有漏洞报告功能。若仓库暂未启用该功能，请先提交一个不包含利用细节的 issue，说明你希望私下报告安全问题，维护者会跟进沟通渠道。

请不要在公开 issue 中发布 token、私钥、可直接利用的 payload、真实用户数据或未修复漏洞的完整利用步骤。

## 关注范围

我们尤其关注：

- 路径遍历和任意文件写入。
- CLI 参数注入、shell 注入和脚本注入。
- Dashboard 中由 Markdown 或项目数据触发的 XSS。
- GitHub Actions 中的 secrets 暴露和不安全触发器。
- npm 包、平台启动器和发布脚本的供应链风险。
- 示例项目或测试 fixture 中意外提交的凭据。

## 预期响应

维护者会尽快确认报告、复现问题并给出修复计划。严重问题修复后，会在发布说明或安全公告中说明影响范围和升级建议。
